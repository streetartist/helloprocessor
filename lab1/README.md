# 实验 1.1：环境搭建与第一个 RISC-V 汇编程序

## 背景知识

RISC-V 是一个开源指令集架构（ISA）。我们学习的是 RV32I —— 32 位基本整数指令集。

关键概念：
- **寄存器**：CPU 内部的高速存储单元，RISC-V 有 32 个通用寄存器（x0-x31）
- **指令**：告诉 CPU 做什么的二进制编码，汇编语言是其人类可读形式
- **系统调用（ecall）**：程序请求操作系统服务（打印、读取、退出等）

## RARS 使用方法

### 启动

```bash
java -jar rars1_6.jar
```

### 界面说明

打开后你会看到：
- **编辑区**（上方）：写汇编代码的地方
- **寄存器面板**（右侧）：实时显示 32 个寄存器的值
- **内存面板**：查看数据段/栈段的内容
- **控制台**（下方）：程序的输入输出

### 操作流程

1. 在编辑区写代码（或 File → Open 打开 .asm 文件）
2. 点击工具栏 **Assemble**（扳手图标）编译
3. 点击 **Run**（播放图标）运行，或 **Step**（单步）逐条执行
4. 观察寄存器和内存的变化

## RISC-V 寄存器一览

| 寄存器 | ABI 名称 | 用途 |
|--------|----------|------|
| x0 | zero | 硬连线为 0，写入无效 |
| x1 | ra | 返回地址 |
| x2 | sp | 栈指针 |
| x5-x7 | t0-t2 | 临时寄存器 |
| x10-x11 | a0-a1 | 函数参数 / 返回值 |
| x10-x17 | a0-a7 | 函数参数 |

完整表见后续实验。现在只需知道：
- 用 `a0-a7` 传参
- 用 `t0-t6` 存临时值

## RARS 系统调用表（常用）

通过 `ecall` 指令触发系统调用，调用号放在 `a7`，参数放在 `a0`/`a1`：

| a7 的值 | 功能 | 参数 |
|---------|------|------|
| 1 | 打印整数 | a0 = 要打印的整数 |
| 4 | 打印字符串 | a0 = 字符串地址 |
| 10 | 退出程序 | 无 |
| 11 | 打印字符 | a0 = ASCII 码 |

## 代码：hello.asm

```asm
# ============================================
# 实验 1.1 - Hello, RISC-V!
# 功能：打印字符串 "Hello, RISC-V!\n"
# ============================================

.data                       # 数据段：存放字符串等静态数据
msg:    .string "Hello, RISC-V!\n"

.text                       # 代码段：存放指令
.globl main                 # 声明 main 为全局入口

main:
    # --- 打印字符串 ---
    li   a7, 4              # 系统调用号 4 = PrintString
    la   a0, msg            # 将字符串地址加载到 a0
    ecall                   # 执行系统调用

    # --- 退出程序 ---
    li   a7, 10             # 系统调用号 10 = Exit
    ecall                   # 执行系统调用
```

### 逐行解释

| 行 | 含义 |
|----|------|
| `.data` | 接下来的内容放到数据段（内存中存数据的区域） |
| `msg: .string "..."` | 定义一个标签 msg，指向这个字符串的内存地址 |
| `.text` | 接下来的内容放到代码段（存指令的区域） |
| `.globl main` | 告诉汇编器 main 是程序入口 |
| `li a7, 4` | Load Immediate —— 把立即数 4 放入寄存器 a7 |
| `la a0, msg` | Load Address —— 把 msg 的地址放入 a0 |
| `ecall` | 触发系统调用，OS 根据 a7 的值决定做什么 |
